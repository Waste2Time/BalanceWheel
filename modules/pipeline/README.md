# 流程编排与串联模块（pipeline）

## 已完成
- `run_pipeline` 端到端示例：无数据时调用数据源 fetcher（可选随机漫步）、交叉验证、写入带元数据的 CSV，再运行简单回测并输出报告。
- `PipelineConfig` 封装标的、频率、路径、时间区间、手续费/资金与主次数据源 fetcher。
- 支持 baseline 与 variant 策略同时回测并生成各自报告。

## 必须完成但未完成
- 更丰富的多策略汇总与实验结果落盘（与 ExperimentRunner/Reporting 聚合）。
- 风控/仓位设置在流水线层面的可配置化（组合策略、再平衡）。
- 自动化的增量回测与重复任务跳过。

## 可选项
- CLI/配置文件入口（YAML/JSON），支持不同场景复用。
- 任务编排与缓存（如增量回测、跳过重复数据下载）。
- 基准/多资产组合的自动汇总报告。
